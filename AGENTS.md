# AGENTS.md

## 项目目标

这是一个“大学回忆视频生成器”。

目标是：

将照片、文字描述、时间信息自动生成具有“青春回忆感”的视频。

整体风格偏向：

- 纪念册
- 青春回忆
- 温暖
- 克制
- 电影感
- 时间流逝感

避免：

- 像 PPT
- 像婚庆模板
- 过度商业化
- 花哨短视频风格
- 高频炫酷特效

---

# 核心功能

## 1. 照片输入

程序需要支持：

- 单张照片
- 多张照片组合
- 横图与竖图混排
- 不同宽高比
- 本地目录读取

照片和视频结构由 YAML 或 JSON 配置文件描述。

---

## 2. 文字描述功能

每张照片可以带文字描述。

支持：

- 标题
- 时间
- 一句话回忆
- 长描述

例如：

```yaml
caption: "第一次参加社团活动"
description: "那时候大家都还不熟。"
```

文字渲染要求：

- 自动换行
- 自动适配宽度
- 保证可读性
- 不超出安全区域
- 支持中文字体

推荐实现：

- 阴影
- 描边
- 半透明背景

避免文字覆盖人脸主体。

---

## 3. 自动布局系统

程序需要自动根据照片数量选择布局。

### 单张照片

支持：

- 全屏铺满
- 留白居中
- Ken Burns 缓慢移动缩放效果

### 两张照片

支持：

- 左右布局
- 上下布局

### 三张照片

推荐：

- 一大两小
- 拼贴布局

### 四张照片

推荐：

- 2x2 网格

### 五张及以上

支持：

- 自动分页
- Masonry 风格
- 动态拼贴

布局要求：

- 不拉伸图片
- 保持比例
- 自动裁切安全区域
- 视觉均衡

---

## 4. 视频风格

默认输出：

- MP4
- 1920x1080
- 30fps

支持：

- Fade in/out
- Crossfade
- 缓慢缩放
- 缓慢平移

转场要求：

- 自然
- 柔和
- 不喧宾夺主

避免：

- 高频闪烁
- 强烈动态特效
- TikTok 风格动画

---

## 5. 时间线逻辑

视频由多个 scene 构成。

每个 scene：

- 可以包含 1~N 张照片
- 可以有标题
- 可以有描述
- 可以单独设置 duration

例如：

```yaml
video:
  title: "大学回忆"
  resolution: [1920, 1080]
  fps: 30

scenes:
  - title: "大一"
    duration: 6
    photos:
      - path: "photos/001.jpg"
        caption: "第一次班会"

      - path: "photos/002.jpg"
        caption: "第一次聚餐"

  - title: "毕业前夜"
    duration: 7
    photos:
      - path: "photos/night.jpg"
        caption: "聊到凌晨"
```

---

## 6. 技术要求

推荐技术栈：

- Python
- MoviePy
- Pillow
- OpenCV
- FFmpeg

要求：

- 使用 type hints
- 模块化
- 易测试
- 避免巨型单文件

推荐模块结构：

```text
src/photo_memory_video/

  cli.py
  config_loader.py
  layout.py
  render.py
  timeline.py
  transitions.py
  text_renderer.py
```

---

## 7. CLI

最终程序需要支持：

```bash
photo-memory-video render \
  --input examples/demo.yaml \
  --output output/demo.mp4
```

---

## 8. 项目优先级

优先级从高到低：

1. 视频氛围感
2. 自动布局质量
3. 中文文字排版
4. 转场自然
5. 渲染性能
6. 高级特效

不要为了复杂特效破坏整体观感。

---

## 9. Definition of Done

满足以下条件才算完成：

- 能从 YAML 成功生成 MP4
- 中文文字正常显示
- 多图布局正常
- README 可运行
- 提供示例配置
- 至少包含基础测试

---

## 10. Agent 工作规则

修改代码前：

1. 阅读项目结构
2. 优先复用已有模块
3. 不随意新增依赖
4. 保持代码风格一致

新增功能时：

- 不破坏已有 CLI
- 不随意修改配置格式
- 优先小步提交

禁止：

- 硬编码路径
- 删除用户数据
- 修改原始图片
- 写不可维护的大函数

---

## 11. MVP 开发顺序

优先实现：

1. YAML 配置读取
2. 自动布局
3. 中文文字渲染
4. MoviePy 视频生成
5. CLI

后期实现：

- 背景音乐自动剪辑
- 智能运镜
- Web UI

先保证：

- 能稳定生成视频
- 代码结构清晰
- 配置格式稳定
